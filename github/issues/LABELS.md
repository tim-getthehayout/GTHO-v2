# GitHub Issue Labels

## Priority
- **P0 — critical** — Blocking other work, needs immediate attention
- **P1 — high** — Should be next in queue
- **P2 — normal** — Standard priority
- **P3 — low** — Nice to have, do when convenient

## Type
- **spec** — Design spec or migration plan
- **decision** — Decision record that needs resolution
- **task** — Actionable work item
- **question** — Needs research or discussion before proceeding

## Area
- **v1-audit** — Related to v1 feature audit and analysis
- **v2-design** — Related to v2 architecture and design decisions
- **migration** — Related to data migration or transition planning
- **infra** — Project infrastructure, tooling, or process
